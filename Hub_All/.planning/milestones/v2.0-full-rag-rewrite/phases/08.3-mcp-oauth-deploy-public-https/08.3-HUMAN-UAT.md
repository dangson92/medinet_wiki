## Tiền điều kiện (operator chuẩn bị TRƯỚC khi test)

1. **Domain công khai** trỏ DNS A record tới IP host chạy MCP Service
   (ví dụ `mcp.medinet.vn`).
2. **Port 80 + 443 mở ra Internet** cho Caddy xin cert ACME (auto-TLS Let's
   Encrypt) — HOẶC dùng **Cloudflare Tunnel** nếu host không mở được port (thay
   service `caddy` bằng `cloudflared`, không cần mở port).
3. Tạo `.env` ở **gốc compose** (`Hub_All/.env`) với 2 biến:
   ```
   MCP_PUBLIC_DOMAIN=mcp.medinet.vn
   MCP_OAUTH_ISSUER_URL=https://mcp.medinet.vn
   ```
   (issuer PHẢI là `https://` domain thật — issuer localhost làm hỏng discovery).
4. **API Service (`python-api`) đang chạy** + có ít nhất **1 tài khoản Medinet**
   (email + password) còn active để đăng nhập trong luồng OAuth.
5. Khởi động stack:
   ```bash
   cd Hub_All
   docker compose up -d
   ```
   Chờ container `medinet-mcp` + `medinet-caddy` ở trạng thái healthy
   (`docker compose ps`).

## Tests

### 1. SC4 — kết nối thật từ Claude web qua "Add custom connector" (human_needed)

expected: thực hiện đủ 7 bước dưới đây, tất cả đạt KỲ VỌNG.

**Bước 1 — OAuth discovery endpoint trả JSON hợp lệ.**
```bash
curl -fsS https://<domain>/.well-known/oauth-authorization-server
```
KỲ VỌNG: HTTP 200 + JSON có các field `issuer`, `authorization_endpoint`,
`token_endpoint`, `registration_endpoint`. Nếu 404 → reverse proxy strip path
sai (Pitfall 7) — kiểm Caddyfile KHÔNG có `handle_path`/`uri strip`.

**Bước 2 — thêm connector trong Claude web.**
Vào Claude web → Settings → Connectors → "Add custom connector" → nhập URL MCP
endpoint `https://<domain>/mcp`.

**Bước 3 — luồng OAuth + login Medinet.**
Claude web mở luồng OAuth → hiển thị form login Medinet → nhập email + password
của tài khoản Medinet → submit.

**Bước 4 — connector kết nối thành công.**
KỲ VỌNG: Claude web báo connector đã kết nối; 3 tool `list_hubs` / `search_wiki`
/ `ask_wiki` xuất hiện trong danh sách tool.

**Bước 5 — gọi `list_hubs`.**
Trong Claude chat, yêu cầu gọi `list_hubs`.
KỲ VỌNG: trả danh sách hub đúng theo role tài khoản — admin → mọi hub active;
user thường → chỉ hub được gán (D-04).

**Bước 6 — gọi `search_wiki` + `ask_wiki`.**
KỲ VỌNG: cả hai tool trả kết quả từ Medinet Wiki (search trả chunk, ask trả câu
trả lời có citation).

**Bước 7 — regression JWT refresh (phiên dài).**
Để connector kết nối > 15 phút (quá TTL JWT downstream 900s) rồi gọi lại một
tool bất kỳ.
KỲ VỌNG: tool vẫn chạy bình thường — downstream JWT đã tự refresh ngầm qua
retry-on-401 (Pitfall 4 / Pitfall 5 — refresh rotation).
