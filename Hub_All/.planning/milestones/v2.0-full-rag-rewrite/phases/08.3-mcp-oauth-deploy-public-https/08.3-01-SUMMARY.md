## Implementation Notes

### Task 1 — Dependencies
- `aiosqlite` resolve **0.22.1** (trong khoảng `>=0.20,<1`), `uvicorn` resolve **0.47.0** (trong khoảng `>=0.30,<1`) — không cần điều chỉnh upper bound.
- `mcp` pin giữ nguyên `>=1.27.0,<1.28`, resolve **1.27.1** (P19 tôn trọng — không bump lên 1.28).
- `uv.lock` sinh ra (chưa tồn tại trước đó) — cần cho `COPY ... uv.lock` của Dockerfile Plan 04.
- KHÔNG thêm package `fastmcp` bên thứ ba (gofastmcp.com — Pitfall 1).
- `uv pip index versions` không khả dụng ở uv 0.11.9 (không có subcommand `index`) → dùng `uv lock` để resolve version thật rồi đọc từ lockfile (xem Deviations).

### Task 2 — Config OAuth
- 4 field mới trên `Settings`: `oauth_issuer_url` (default `http://localhost:8190`), `oauth_state_db_path` (default `.oauth/state.db`), `oauth_access_token_ttl` (3600), `oauth_refresh_token_ttl` (2592000).
- `_validate_issuer_url` (mode=after) ép scheme http/https + bắt buộc host — verify: `ftp://x` và `https://` (thiếu host) đều reject exit≠0; default chạy exit 0.
- `oauth_issuer_url` có default cố ý → `Settings()` không raise khi thiếu env (well-known route trả 200 cho HEALTHCHECK Plan 04).

### Task 3 — OAuthStore + test scaffolding
- `OAuthStore` quản 4 bảng: `oauth_clients`, `oauth_auth_codes`, `oauth_tokens`, `oauth_pending` — schema khởi tạo idempotent qua `CREATE TABLE IF NOT EXISTS`.
- CRUD roundtrip đầy đủ: `save_client/get_client`, `save_auth_code/load_auth_code/delete_auth_code`, `save_token/load_token/load_token_by_refresh/delete_token`, `save_pending/load_pending/delete_pending`.
- `update_downstream_jwt` ghi đè CẢ `downstream_jwt` + `downstream_refresh_token` (Pitfall 5 — refresh rotation), `update_oauth_token` rotate token OAuth.
- `cleanup_expired` dọn auth code hết hạn + pending bỏ dở >600s.
- Mọi query dùng placeholder `?` (parametrized — T-08.3-04). Không log giá trị secret, chỉ log `client_id` (T-08.3-01).
- Quyết định thiết kế: giữ **một connection** mở sau `init_schema()` → fixture test dùng SQLite `:memory:` chạy được (đóng connection = mất DB).
- conftest thêm fixture `oauth_store` (:memory:) + helper `async fake_pending_authorize` seed `oauth_pending` cho Plan 02/03.
- `test_oauth_store.py`: 10 test, `test_token_roundtrip` + `test_update_downstream_jwt_rotates_both` đánh `@pytest.mark.critical`.
