## Tóm tắt

Phase 8.1 dựng một MCP server (Model Context Protocol) expose 3 tool read-only (`search_wiki`, `ask_wiki`, `list_hubs`) lên trên service layer RAG sẵn có của Medinet Wiki. MCP server được mount vào FastAPI app chính tại path `/mcp` theo kiến trúc in-process (Pattern 1 — ARCHITECTURE.md), tái dùng `app.state.db_pool`, `app.state.redis`, và toàn bộ service layer đã implement từ Phase 6/7.

**Primary recommendation:** Dùng `FastMCP` (package `mcp`) + `http_app(stateless_http=True)` + `combine_lifespans()` mount tại `/mcp`. Auth qua `get_http_headers(include={'x-api-key'})` trong tool handler → gọi `ApiKeyService.verify_key` → dựng `UserWithHubs` thủ công. Tool trả Pydantic model → FastMCP tự generate structured output.

**Quan trọng — Landmine lifespan (xem §Cạm bẫy thường gặp):** FastMCP dùng session manager (task group) PHẢI được khởi động trong lifespan của ứng dụng ASGI cha. KHÔNG mount `mcp_app` vào FastAPI mà không pass lifespan — sẽ gặp lỗi `Task group is not initialized` khi client gọi.

### Pattern 1: Mount FastMCP vào FastAPI — Lifespan Composition (CRITICAL)

**Vấn đề cốt lõi:** FastMCP Streamable HTTP dùng session manager (task group) cần được khởi động trong lifespan của ASGI app cha. Nếu chỉ `app.mount("/mcp", mcp_app)` mà không pass lifespan → lỗi runtime `Task group is not initialized. Make sure to use run()`.

**Giải pháp (từ docs FastMCP + verified issue #1367):**

```python
# api/app/mcp/server.py
from mcp.fastmcp import FastMCP

mcp = FastMCP("Medinet Wiki MCP Server")

# Tool definitions tại đây (xem Pattern 2)
```

```python
# api/app/main.py  — MODIFIED

# BƯỚC 1: Import mcp server TRƯỚC khi create_app() (lazy import để tránh
# circular import với app.state)
from app.mcp.server import mcp as _mcp_server

# BƯỚC 2: Tạo MCP ASGI app với stateless_http=True
# stateless_http=True: mỗi request là fresh context → không cần session affinity,
# phù hợp với single-process uvicorn M2 (KHÔNG cần sticky session).
_mcp_app = _mcp_server.http_app(path="/", stateless_http=True)

# BƯỚC 3: Combine lifespans
from fastmcp.utilities.lifespan import combine_lifespans

@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncIterator[None]:
    # ... existing lifespan code unchanged ...
    # db_pool, redis, cocoindex, jwt_manager, watchdog, audit_flush, search_cache
    ...
    yield
    # ... shutdown unchanged ...

# combine_lifespans: enters in order → exits in REVERSE order
app = FastAPI(
    lifespan=combine_lifespans(lifespan, _mcp_app.lifespan),
    ...
)

# BƯỚC 4: Mount tại /mcp
app.mount("/mcp", _mcp_app)
```

**Source:** [FastMCP FastAPI integration docs](https://gofastmcp.com/integrations/fastapi) [CITED] + [Issue #1367](https://github.com/modelcontextprotocol/python-sdk/issues/1367) [CITED]

**Tại sao `stateless_http=True`:**
- M2 chạy single uvicorn process, 1 worker — không cần sticky session.
- Tránh vấn đề session state memory leak trong long-running process.
- Phù hợp cho tool call đơn giản (read-only, stateless by nature).

**Tại sao `path="/"`:**
Khi mount tại `app.mount("/mcp", mcp_app)`, FastMCP nhận request với path stripped về `/`. Truyền `path="/"` (KHÔNG `path="/mcp"`) để tránh double-prefix.

[VERIFIED: pattern từ FastMCP docs — `mcp_app = mcp.http_app(path='/')` + `app.mount("/mcp", mcp_app)`]

### Pattern 2: Đọc HTTP header trong tool handler — `get_http_headers`

**Mục đích:** Lấy `X-API-Key` từ HTTP request trong tool handler (D-05, D-12).

```python
# api/app/mcp/auth.py
from __future__ import annotations

import asyncpg
from fastmcp.server.dependencies import get_http_headers

from app.auth.dependencies import UserWithHubs
from app.models.auth import User, UserHub
from app.services.api_key_service import ApiKeyService
from sqlalchemy import select, text


async def authenticate_mcp_request(pool: asyncpg.Pool) -> UserWithHubs:
    """Đọc X-API-Key từ HTTP header → verify → dựng UserWithHubs.

    Gọi từ đầu mỗi tool handler (KHÔNG Depends — không có FastAPI DI trong MCP).
    Raise ValueError với message rõ ràng nếu auth fail → tool handler map sang
    FastMCP ToolError → client nhận MCP error.

    Cần AsyncSession để gọi ApiKeyService.verify_key (SQLAlchemy-based).
    Dùng get_session() context manager hoặc inject qua pool + sqlalchemy engine.
    """
    headers = get_http_headers(include={"x-api-key"})
    api_key = headers.get("x-api-key")
    if not api_key:
        raise ValueError("MCP_UNAUTHORIZED: thiếu header X-API-Key")

    # Tạo async session tạm để verify key
    from app.db.session import get_engine
    from sqlalchemy.ext.asyncio import AsyncSession
    engine = get_engine()
    async with AsyncSession(engine) as db:
        svc = ApiKeyService(db=db)
        principal = await svc.verify_key(api_key)
        if principal is None:
            raise ValueError("MCP_UNAUTHORIZED: API key không hợp lệ hoặc đã thu hồi")

        # Load created_by user
        key_row = (
            await db.execute(
                text("SELECT created_by FROM api_keys WHERE id = :id"),
                {"id": principal["id"]},
            )
        ).fetchone()
        if key_row is None or key_row[0] is None:
            raise ValueError("MCP_UNAUTHORIZED: API key không gắn user hợp lệ")

        stmt = select(User).where(User.id == key_row[0], User.is_active.is_(True))
        user = (await db.execute(stmt)).scalar_one_or_none()
        if user is None:
            raise ValueError("MCP_UNAUTHORIZED: User gắn API key đã bị vô hiệu hoá")

        # Load hub_ids (giống get_current_user_with_hubs trong dependencies.py)
        stmt_hubs = select(UserHub.hub_id).where(UserHub.user_id == user.id)
        hub_ids = [str(h) for h in (await db.execute(stmt_hubs)).scalars().all()]
        await db.commit()  # commit SAU background work (pattern: project_fastapi_bgtask_commit.md)

    return UserWithHubs(user=user, hub_ids=hub_ids)
```

**Lưu ý quan trọng từ issue FastMCP #1233 [CITED]:** Khi dùng `stateless_http=True`, `get_http_headers()` trả headers của request hiện tại (KHÔNG bị stale). Vấn đề stale headers CHỈ xảy ra khi có session persistent (stateful mode, cùng session nhiều requests). Với `stateless_http=True` (mode của chúng ta), mỗi request là fresh context → `get_http_headers()` luôn đúng.

### Pattern 4: Truy cập `app.state.db_pool` từ MCP tool handler

**Vấn đề:** MCP tool handler KHÔNG có FastAPI `Request` object → không dùng `request.app.state`.

**Giải pháp:** Dùng module-level singleton (pattern tương tự `get_engine()` trong `app/db/session.py`):

```python
# api/app/mcp/server.py
from __future__ import annotations
import asyncpg
from typing import Any

# Module-level pool được set ở lifespan TRƯỚC khi tool nào được gọi
_pool: asyncpg.Pool | None = None
_redis: Any = None


def set_pool(pool: asyncpg.Pool, redis: Any = None) -> None:
    """Gọi từ lifespan FastAPI sau khi db_pool init."""
    global _pool, _redis
    _pool = pool
    _redis = redis


def _get_pool() -> asyncpg.Pool:
    if _pool is None:
        raise RuntimeError("db_pool chưa sẵn sàng — MCP không khả dụng")
    return _pool


def _get_redis() -> Any:
    return _redis  # None OK — SearchService/AskService xử lý redis=None
```

```python
# api/app/main.py — trong lifespan, sau bước 1 (db_pool init):
from app.mcp.server import set_pool as mcp_set_pool

# Trong lifespan, sau khi app.state.db_pool được gán:
if app.state.db_pool is not None:
    mcp_set_pool(app.state.db_pool, app.state.redis)
```

**Tại sao không dùng `get_http_request().app.state`:** `get_http_request()` trả Starlette `Request` của MCP sub-app, KHÔNG có `app.state` của FastAPI cha. Module-level singleton là pattern an toàn nhất.

### Pattern 5: Tool definition hoàn chỉnh — `ask_wiki`

```python
# api/app/mcp/server.py (minh họa ask_wiki)
from fastmcp.exceptions import ToolError
from app.mcp.schemas import AskAnswer, CitationItem
from app.schemas.ask import AskRequest
from app.services.ask_service import AskError, AskService
from app.services.usage_service import log_usage_event
import asyncio

@mcp.tool
async def ask_wiki(
    query: str,
    hub_id: str | None = None,
) -> AskAnswer:
    """Hỏi-đáp RAG từ Medinet Wiki. Trả câu trả lời + danh sách trích dẫn có số [N].

    Args:
        query: Câu hỏi bằng tiếng Việt hoặc tiếng Anh.
        hub_id: ID Hub cụ thể (tuỳ chọn). Để trống = hỏi cross-hub mọi Hub được phép.

    Returns:
        answer: Câu trả lời với marker [1], [2]... chỉ nguồn trích dẫn.
        citations: Danh sách nguồn tương ứng với marker [N].
    """
    pool = _get_pool()
    try:
        user = await authenticate_mcp_request(pool)
    except ValueError as e:
        raise ToolError(str(e))

    ask_svc = AskService(pool=pool, redis=_get_redis())
    body = AskRequest(query=query, hub_id=hub_id)
    cross_hub = hub_id is None

    try:
        if cross_hub:
            outcome = await ask_svc.ask_cross_hub(body=body, user=user)
        else:
            outcome = await ask_svc.ask(body=body, user=user)
    except AskError as e:
        raise ToolError(f"LLM lỗi: {e}")
    except Exception as e:
        raise ToolError(f"Lỗi nội bộ: {e}")

    # D-14: log usage_events non-blocking
    asyncio.create_task(
        log_usage_event(
            pool,
            user_id=str(user.user.id),
            hub_id=hub_id,
            model=outcome.usage.model,
            prompt_tokens=outcome.usage.prompt_tokens,
            completion_tokens=outcome.usage.completion_tokens,
            total_tokens=outcome.usage.total_tokens,
            cost_usd=outcome.usage.cost_usd,
            request_id=None,
        )
    )

        answer=outcome.response.answer,
        citations=[
            CitationItem(
                chunk_id=c.chunk_id,
                document_name=c.document_name,
                hub_name=c.hub_name,
                snippet=c.content_snippet,
                score=c.score,
            )
            for c in outcome.response.citations
        ],
    )
```

### Anti-patterns cần tránh

- **KHÔNG** mount MCP sub-app mà không pass lifespan → `Task group is not initialized`.
- **KHÔNG** dùng `stateful` mode (default) với nhiều uvicorn workers — session state bị mất.
- **KHÔNG** self-call HTTP `/api/*` từ tool (D-04) — tăng latency + bypass auth isolation.
- **KHÔNG** tạo asyncpg pool mới trong tool — tái dùng pool từ `app.state.db_pool` qua module singleton.
- **KHÔNG** dùng `FastAPI.Depends` trong MCP tool handler — không có DI context.
- **KHÔNG** dùng `get_http_request().app.state` để lấy db_pool — sub-app không có state của cha.

## Đừng tự viết (Don't Hand-Roll)

| Vấn đề | Đừng tự xây | Dùng thay | Lý do |
|--------|------------|----------|-------|
| MCP JSON-RPC protocol | Custom HTTP handler | `mcp` SDK (FastMCP) | Protocol complex: initialize, tool_call, error format, content blocks |
| Structured tool output schema | Viết JSON Schema tay | FastMCP auto từ Pydantic model | FastMCP tự generate `outputSchema` từ type hint |
| Streamable HTTP session | Custom SSE handler | `mcp.http_app(stateless_http=True)` | Session manager đã handle reconnect, heartbeat, content-type |
| Auth middleware MCP | Custom ASGI middleware | `get_http_headers()` trong tool handler | Đơn giản hơn, test dễ hơn, tái dùng `ApiKeyService` sẵn |
| ToolError format | Raise Exception thường | `from fastmcp.exceptions import ToolError` | FastMCP format đúng MCP error JSON-RPC (`isError: true`) |
| Lifespan composition | Viết nested lifespan | `combine_lifespans()` từ `fastmcp.utilities.lifespan` | Enters in order, exits in reverse — pattern đã verified |

## Cạm bẫy thường gặp (Common Pitfalls)

### Cạm bẫy 1: Lifespan session manager không khởi động (BLOCKER)

**Xảy ra khi nào:** Mount `mcp_app` vào FastAPI mà KHÔNG pass lifespan.
```python
# SAI — session manager không init:
app = FastAPI(lifespan=existing_lifespan)
app.mount("/mcp", mcp_app)  # mcp_app.lifespan KHÔNG được gọi
```
**Lỗi runtime:** `Task group is not initialized. Make sure to use run()`
**Cách tránh:** Luôn dùng `combine_lifespans(existing_lifespan, mcp_app.lifespan)`.
**Nguồn:** [CITED: FastMCP docs + modelcontextprotocol/python-sdk#1367]

### Cạm bẫy 2: Double path prefix

**Xảy ra khi nào:** `mcp.http_app(path="/mcp")` + `app.mount("/mcp", mcp_app)`.
**Lỗi:** Client nhận redirect `/mcp → /mcp/` + 404 hoặc path mismatch.
**Cách tránh:** `mcp.http_app(path="/")` khi mount tại `/mcp`.

### Cạm bẫy 3: `get_http_headers()` stale khi stateful mode

**Xảy ra khi nào:** Không dùng `stateless_http=True`, cùng MCP session nhiều request.
**Lỗi:** Tool handler nhận headers của request đầu tiên, không phải request hiện tại.
**Cách tránh:** `mcp.http_app(stateless_http=True)` — mỗi request fresh context.
**Nguồn:** [CITED: PrefectHQ/fastmcp#1233]

### Cạm bẫy 4: circular import `app.mcp.server` ↔ `app.main`

**Xảy ra khi nào:** `server.py` import từ `main.py` (ví dụ `from app.main import app`).
**Cách tránh:** Module singleton (`_pool`, `set_pool`) + `from app.db.session import get_engine` thay vì import `app`. `main.py` import `mcp.server`, KHÔNG ngược lại.

### Cạm bẫy 5: asyncpg pool KHÔNG có trong MCP module khi tool call đầu tiên

**Xảy ra khi nào:** `set_pool()` gọi trễ hơn lifespan bình thường, hoặc pool init fail.
**Lỗi:** `RuntimeError: db_pool chưa sẵn sàng`
**Cách tránh:** `set_pool()` gọi ngay sau khi `app.state.db_pool` được gán trong lifespan (bước 1). `combine_lifespans` đảm bảo lifespan cha chạy TRƯỚC lifespan MCP — đủ thời gian init pool.

### Cạm bẫy 6: `commit()` trong `authenticate_mcp_request` sau verify_key

**Xảy ra khi nào:** `ApiKeyService.verify_key` gọi `UPDATE api_keys SET last_used_at = NOW()` (background writes) — nếu không `commit()` trước khi session close → UPDATE bị rollback.
**Cách tránh:** Thêm `await db.commit()` sau `verify_key` + hub_ids load. (Pattern: `project_fastapi_bgtask_commit.md` trong MEMORY.md)

### Cạm bẫy 7: Rate limit MCP tool call

**Xảy ra khi nào:** slowapi chạy trên `@router.post(...)` — KHÔNG áp dụng cho ASGI sub-app `/mcp`.
**Kết luận:** MCP tool call KHÔNG bị rate limit slowapi hiện tại. Nếu cần → implement custom trong `authenticate_mcp_request` (Redis counter theo `user_id`). [Claude's Discretion — tự quyết]

## Ví dụ code (Code Examples)

### Ví dụ 1: `list_hubs` tool (đơn giản nhất — kiểm tra auth trước)

```python
# api/app/mcp/server.py
@mcp.tool
async def list_hubs() -> HubList:
    """Liệt kê các Hub Medinet Wiki mà API key có quyền truy cập.
    Dùng hub_id để filter trong search_wiki / ask_wiki.
    """
    pool = _get_pool()
    try:
        user = await authenticate_mcp_request(pool)
    except ValueError as e:
        raise ToolError(str(e))

    # Query hubs từ user_hubs join
    async with pool.acquire() as conn:
        if user.user.role == "admin":
            rows = await conn.fetch(
                "SELECT id, name, description FROM hubs WHERE status = 'active'"
            )
        else:
            rows = await conn.fetch(
                "SELECT h.id, h.name, h.description FROM hubs h "
                "JOIN user_hubs uh ON h.id = uh.hub_id "
                "WHERE uh.user_id = $1 AND h.status = 'active'",
                user.user.id,
            )
    return HubList(
        hubs=[HubItem(id=str(r["id"]), name=r["name"], description=r["description"])
              for r in rows]
    )
```

### Ví dụ 2: `search_wiki` tool — single/cross-hub routing (D-08)

```python
@mcp.tool
async def search_wiki(
    query: str,
    hub_id: str | None = None,
    top_k: int = 10,
) -> SearchResult:
    """Tìm kiếm tri thức Medinet Wiki bằng vector semantic search.

    Args:
        query: Câu truy vấn tìm kiếm.
        hub_id: ID Hub cụ thể (tuỳ chọn). Để trống = cross-hub.
        top_k: Số chunk trả về (mặc định 10, tối đa 20).
    """
    pool = _get_pool()
    try:
        user = await authenticate_mcp_request(pool)
    except ValueError as e:
        raise ToolError(str(e))

    from app.services.search_service import SearchService
    from app.schemas.search import SearchRequest
    from app.repositories.hub_isolation import HubIsolationError

    svc = SearchService(pool=pool, redis=_get_redis())
    top_k = max(1, min(top_k, 20))  # clamp

    try:
        if hub_id:
            result = await svc.search(
                body=SearchRequest(query=query, hub_ids=[hub_id], top_k=top_k),
                user=user,
            )
        else:
            result = await svc.search_cross_hub(
                body=SearchRequest(query=query, hub_ids=None, top_k=top_k),
                user=user,
            )
    except HubIsolationError as e:
        raise ToolError(f"HUB_ACCESS_DENIED: {e}")
    except Exception as e:
        raise ToolError(f"Search lỗi: {e}")

    items = [
        SearchResultItem(
            chunk_id=str(r["id"]),
            content=r["snippet"] or r.get("content", ""),
            score=float(r["score"]),
            document_name=r["title"],
            hub_name=r["hub_name"],
            hub_id=str(r["hub_id"]),
        )
        for r in result["results"]
    ]
    return SearchResult(results=items, total=len(items))
```

## Security Domain

### Applicable ASVS Categories

| ASVS Category | Áp dụng | Standard Control |
|---------------|---------|-----------------|
| V2 Authentication | yes | `ApiKeyService.verify_key` (Phase 5 AUX-02 — đã implement) |
| V3 Session Management | không | `stateless_http=True` — KHÔNG có session |
| V4 Access Control | yes | `UserWithHubs` + `HubIsolationError` (Phase 5 HUB-02) |
| V5 Input Validation | yes | Pydantic type hints trên tool params — FastMCP validate tự động |
| V6 Cryptography | không liên quan | API key verify qua AES-GCM (đã có từ AUX-02) |

### Known Threat Patterns cho MCP/HTTP stack

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| API key không có trong request | Spoofing | `get_http_headers()` → check None → `ToolError("MCP_UNAUTHORIZED")` |
| API key đã thu hồi vẫn gửi | Spoofing | `ApiKeyService.verify_key` check `is_active=TRUE` |
| Client khai `hub_id` ngoài quyền | Elevation | `HubIsolationError` ở service layer → `ToolError("HUB_ACCESS_DENIED")` |
| Tool call bypass auth | Elevation | Auth check đầu mỗi tool handler — KHÔNG trust tool params |
| Thông tin lỗi nội bộ leak | Information Disclosure | `ToolError(message)` — KHÔNG expose stack trace; FastMCP `mask_error_details` option |
| Flood tool calls | Denial of Service | Cân nhắc Redis counter trong `authenticate_mcp_request` [Claude's Discretion] |

**Lưu ý bảo mật quan trọng:** Header `X-API-Key` đi qua HTTP plain text khi test local. Production PHẢI có TLS (uvicorn + nginx hoặc `uvicorn --ssl-*`). Phase 8.1 scope = local dev + internal use → defer TLS setup v4.0 hardening.

## RESEARCH COMPLETE

**Phase:** 8.1 - MCP Server — Expose Wiki Tools
**Confidence:** HIGH

### Key Findings

1. **Version pin an toàn:** `mcp==1.9.4` tương thích `fastapi==0.136.1` + Python 3.12 — KHÔNG xung đột dependency (starlette 1.0.0 được resolve đúng bởi cả hai).

2. **Landmine lifespan (CRITICAL):** FastMCP Streamable HTTP PHẢI có lifespan session manager chạy. Dùng `combine_lifespans(existing_lifespan, mcp_app.lifespan)` khi tạo FastAPI app — KHÔNG chỉ `app.mount()`.

3. **Header auth khả thi với `stateless_http=True`:** `get_http_headers(include={'x-api-key'})` trả headers đúng của mỗi request khi dùng stateless mode (stale headers bug CHỈ xảy ra trong stateful mode).
