# Phase 8.2 Plan 02: Mở auth API Service cho MCP Service (X-API-Key) Summary

Mở auth API Service để MCP Service (gọi qua HTTP, forward `X-API-Key` của client) dùng được toàn bộ 4 endpoint mà 3 tool MCP cần — thêm dependency `get_api_key_or_jwt_with_hubs` (verify X-API-Key HOặC JWT → load `hub_ids` từ `user_hubs` → trả `UserWithHubs`), đổi 4 endpoint search/ask (+`/api/ask/cross-hub` đồng bộ) sang dependency này, nới `GET /api/hubs` sang `get_api_key_or_jwt` + scope theo role qua `HubService.list_for_hubs`. Đây là BLOCKER fix điều kiện tiên quyết cho Plan 03 (SC2/SC3/SC4).

## Điểm chính

- **`get_api_key_or_jwt_with_hubs`** (`dependencies.py`): analog `get_current_user_with_hubs` nhưng nhận `user` qua `Depends(get_api_key_or_jwt)` thay vì `Depends(get_current_user)` — nhờ vậy đường X-API-Key được verify trước (qua `ApiKeyService.verify_key`), đường JWT vẫn fallback. `hub_ids` load từ DB `user_hubs` (`select(UserHub.hub_id).where(UserHub.user_id == user.id)`) — KHÔNG tin payload. `get_current_user`, `get_current_user_with_hubs`, `get_api_key_or_jwt`, `require_role` KHÔNG đổi thân hàm — chỉ thêm hàm mới.
- **4 endpoint search/ask** (`search.py` 3 endpoint, `ask.py` `/api/ask` + `/api/search/answer`): đổi `Depends(get_current_user_with_hubs)` → `Depends(get_api_key_or_jwt_with_hubs)`. Type annotation `UserWithHubs` giữ nguyên — service layer không đổi. `/api/ask/cross-hub` cũng đổi đồng bộ (MCP `ask_wiki` cross-hub thực tế gọi `/api/search/answer`, không gọi endpoint này — nhưng đổi cho nhất quán auth path).
- **`GET /api/hubs`** (`hubs.py`): đổi `Depends(require_role("admin"))` → `Depends(get_api_key_or_jwt)` + thêm `db` injection. `user.role == "admin"` → `HubService.list()` (mọi hub); non-admin → load `hub_ids` từ `user_hubs` → `HubService.list_for_hubs()`. 5 endpoint hubs mutate (POST/PUT/PATCH/stats) GIỮ NGUYÊN `require_role("admin")`.
- **`HubService.list_for_hubs`** (`hub_service.py`): analog `list()` nhưng thêm `WHERE id = ANY(:hub_ids)` named bind param (asyncpg parametrize). Guard `hub_ids` rỗng → trả `([], 0)` NGAY (tránh SQL `ANY([])`).
- **Usage logging (SC4):** `/api/ask` `background_tasks.add_task(log_usage_event, ...)` GIỮ NGUYÊN — không phụ thuộc kiểu auth. Test 6 verify `add_task` được gọi với `log_usage_event` qua X-API-Key path.
