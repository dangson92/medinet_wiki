Tạo `api/scripts/smoke/__init__.py` (file rỗng, package marker) và `api/scripts/smoke/contract_diff.py`.

`contract_diff.py` là script Python standalone (chạy `python -m scripts.smoke.contract_diff` từ thư mục `api/`, hoặc `python api/scripts/smoke/contract_diff.py` từ `Hub_All/`). Logic:

1. **Trích endpoint frontend:** Đọc `frontend/src/services/api.ts` (đường dẫn tương đối tính từ vị trí script — dùng `pathlib.Path(__file__).resolve().parents[3] / "frontend/src/services/api.ts"`). Regex bắt mọi lời gọi `this.request<...>('METHOD', '/api/...')` và `fetch(\`${this.baseURL}/api/...\`, { method: 'METHOD' })`. Pattern gợi ý: `r"""this\.request<[^>]*>\(\s*'(\w+)',\s*[`']([^`']+)[`']"""` cho request, và `r"""fetch\(`\$\{this\.baseURL\}([^`]+)`,\s*\{\s*method:\s*'(\w+)'"""` cho fetch trực tiếp (upload/reupload). Path template hoá: thay `${...}` interpolation thành `{id}`. Kết quả: set các tuple `(METHOD, path_template)`.

2. **Trích endpoint FastAPI:** Đọc 10 file router trong `api/app/routers/` + `api/app/auth/router.py`. Với mỗi file lấy `prefix=` trong `APIRouter(...)` rồi ghép với path trong decorator `@router.<method>("...")`. Regex: `r'APIRouter\([^)]*prefix=[\'"]([^\'"]+)[\'"]'` cho prefix, `r'@router\.(get|post|put|delete|patch)\(\s*[\'"]([^\'"]*)[\'"]'` cho route. Normalize `{document_id}`/`{hub_id}`/`{id}` về `{id}`. Kết quả: set các tuple `(METHOD, path)`.

3. **Diff + phân loại:** Với mỗi endpoint frontend KHÔNG khớp FastAPI, gán nhãn theo bảng phân loại cứng (hardcode dict trong script — đây là quyết định planner, executor KHÔNG tự suy diễn):
   - `/api/ai/chat` → `"BLOCKER"` (GeminiAssistant trên golden path Dashboard)
   - `/api/sync/` (prefix match) → `"EXCLUDED"` lý do `"sync queue loại khỏi M2 — Phase 5 CONTEXT D-01"`
   - `/api/documents/{id}/versions`, `/reupload`, `/content` (substring match) → `"EXCLUDED"` lý do `"version history defer v4.1 — RAG-V4-03"`
   - `/api/hubs/{id}/test-connection` → `"EXCLUDED"` lý do `"D-06 PROJECT.md — KHONG test-connection"`
   - `/api/documents/compose` → `"FIX-API"` lý do `"compose mode DocumentIngestion — cần stub api-side"`
   - `/api/documents/{id}/file` → `"FIX-API"` lý do `"tải file gốc — cần stub api-side"`
   - mặc định → `"UNCLASSIFIED"` (script in cảnh báo nếu có endpoint rơi vào đây)

4. **Output:** In ra stdout bảng phẳng 3 cột `METHOD | PATH | STATUS(MATCH/BLOCKER/EXCLUDED/FIX-API/UNCLASSIFIED) | REASON`, đồng thời ghi file Markdown `.planning/phases/08-frontend-e2e-smoke/08-CONTRACT-DIFF.md`. Exit code: 0 nếu KHÔNG có `UNCLASSIFIED`, 1 nếu có (để CI/executor phát hiện endpoint mới chưa phân loại).

Không cần dependency ngoài (chỉ `re`, `pathlib`, `sys`). Tuân `mypy --strict`: annotate mọi hàm trả `list[tuple[str, str]]` / `dict[tuple[str,str], str]`. Tuân `ruff`: không import thừa.
