1. **Mục đích** — Eval dataset + baseline retrieval cho M1 RAG Quality. Phase 1 baseline native, Phase 5 compare với Docling.

2. **Yêu cầu môi trường**:
   - Python ≥ 3.11
   - Backend Go đang chạy ở `http://localhost:8180`
   - Postgres + Redis + ChromaDB đang chạy (theo `backend/.env.example`)
   - LibreOffice headless (cho `build_scanned.py`) — Windows: cài LibreOffice → thêm `soffice.exe` vào PATH; fallback `docx2pdf` (yêu cầu MS Word) hoặc `pandoc + wkhtmltopdf`.

3. **Cài đặt**:
   ```bash
   cd eval
   python -m venv .venv
   source .venv/bin/activate  # Windows: .venv\Scripts\activate
   pip install -e ".[dev]"
   cp .env.example .env
   # Sửa .env theo môi trường local
   ```

4. **Cấu trúc thư mục** — copy nguyên block từ 01-CONTEXT.md mục "Cấu trúc thư mục eval/".

5. **Workflow build dataset (chạy 1 lần)**:
   ```bash
   # Step 1: Tạo 2 scanned PDF từ DOCX gốc
   python eval/scripts/build_scanned.py

   # Step 2: Extract heading vàng từ DOCX
   python eval/scripts/extract_headings.py

   # Step 3: Sinh draft 12 queries (LLM-driven, output đề xuất → user review)
   #   Hiện tại file queries.jsonl đã commit; nếu cần regen, xem 01-05-PLAN.md.
   ```

6. **Workflow chạy baseline native (Phase 1)**:
   ```bash
   # Step 1: Seed eval_hub vào Postgres (chạy 1 lần)
   psql -h localhost -U medinet -d medinet_central -f eval/scripts/seed_hub.sql

   # Step 2: Cleanup state cũ (idempotent — chạy mỗi lần trước baseline)
   python eval/scripts/cleanup.py

   # Step 3: Restart backend Go (để re-create ChromaDB collection nếu cần)
   #   HOẶC cleanup.py tự re-create collection qua ChromaDB API.

   # Step 4: Chạy baseline — upload all + measure retrieval
   python eval/baseline.py
   #   Output: eval/baseline_native.json
   ```
