  **app/services/watchdog.py (NEW):**
  - `WATCHDOG_INTERVAL_SECONDS = 60` const.
  - `def get_watchdog_timeout_seconds() -> int` — return `get_settings().watchdog_timeout_seconds` (tách để test mock).
  - `async def watchdog_tick() -> int` — mở session từ engine; chạy:
    ```sql
    UPDATE documents SET status='failed',
      error_message='timeout: no heartbeat for >' || :timeout_secs || 's',
      updated_at=NOW()
    WHERE status='processing'
      AND last_heartbeat IS NOT NULL
      AND last_heartbeat < NOW() - make_interval(secs => :timeout_secs)
    ```
    Bind `:timeout_secs` = `get_watchdog_timeout_seconds()`. Trả `result.rowcount`. `logger.info` nếu count > 0. Try/except RuntimeError (engine chưa init / đã dispose) → skip + log warning.
  - `async def watchdog_loop() -> None` — forever loop: `await watchdog_tick()` rồi `await asyncio.sleep(WATCHDOG_INTERVAL_SECONDS)`. Cancel-safe: `except asyncio.CancelledError: return`. Exception khác → `logger.exception` + continue (KHÔNG crash task).

  **app/main.py — lifespan APPEND-ONLY:**
  - Startup, SAU step cocoindex setup + engine init: `app.state.watchdog_task = asyncio.create_task(watchdog_loop())` + log "watchdog_task_started". Try/except → log warning nếu fail.
  - Shutdown, đầu finally TRƯỚC dispose_engine: `app.state.watchdog_task.cancel()` + `await app.state.watchdog_task` suppress `asyncio.CancelledError`. Cancel TRƯỚC dispose_engine để tránh race "engine disposed mid-tick".
  </action>
