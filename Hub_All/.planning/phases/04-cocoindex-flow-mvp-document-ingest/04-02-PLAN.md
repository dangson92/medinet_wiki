# Plan 04-02: Services file_extract + vn_chunker + embedder + file_store

<objective>
Tách lớp services độc lập với cocoindex flow (Plan 04-03 sẽ wrap chúng qua `@coco.fn`). Các service đều **char-based + provider-agnostic** để cocoindex flow chỉ là glue layer. Bốn module:

1. **file_extract.py** — Extract text 4 format: DOCX (python-docx), TXT/MD (UTF-8 plain), PDF (pypdf text-only + scanned detect). R4 mitigation: PDF scanned → `is_scanned=True` để router Plan 04-04 set `status='failed_unsupported'`. `detect_scanned_pdf` là hàm PUBLIC riêng cho router gọi synchronous trước INSERT.
2. **vn_chunker.py** — Custom regex split theo heading VN (`Mục N.`, `Chương N.`, `\d+\.`), blank line, sentence end before VN caps (P13). Output `ChunkDraft` dataclass. Char-based size (P14 cross-provider — KHÔNG token-based).
3. **embedder.py** — Wrap `litellm.aembedding(dimensions=1536)` (R1 mitigation pin dim 1536 cho hot-swap OpenAI ↔ Gemini WITHIN dim). Hot-swap qua `settings.rag_embedding_model`. Export `EMBEDDING_DIM=1536`.
4. **file_store.py** — `FileStore` class local backend (`file_store/<uuid>.<ext>`). `ALLOWED_EXTENSIONS` frozenset + `UnsupportedFormatError`.

Purpose: Plan 04-03 flow wrap services qua `@coco.fn`; Plan 04-04 router gọi FileStore.save + detect_scanned_pdf; Phase 6/7 search/ask gọi embed_text độc lập (KHÔNG phụ thuộc cocoindex context).
Output: 4 service module + 4 unit test file.
</objective>

    2. Split theo heading patterns thành section, mỗi section giữ heading line làm `heading_path`.

    4. **QUAN TRỌNG (gap SC2):** input có nội dung text → output PHẢI có >= 1 ChunkDraft. KHÔNG bao giờ trả [] cho text non-empty. Nếu không match heading nào → cả text là 1 section "no heading".
  - Char-based (KHÔNG token-based) — P14 provider-agnostic.
