# Plan 04-04: Documents router POST /upload + GET /:id + A4 BackgroundTasks

<objective>
Hiện thực INGEST-04 (upload endpoint) + INGEST-05 (GET status). Thay LISTEN/NOTIFY (cocoindex 1.0.3 KHÔNG hỗ trợ source notification) bằng **A4 BackgroundTasks pattern** — router add task `trigger_cocoindex_update` chạy `cocoindex_app.update_blocking()` ngay sau response 202, giữ SLA <5s.

**Bài học gap zero-chunks (04-VERIFICATION.md):** Lần execute trước SC2 fail vì cocoindex flow generate 0 chunks. Một root-cause đã xác định: nếu `service.create` để `get_session()` auto-commit (commit SAU khi BackgroundTask chạy), thì `cocoindex_app.update_blocking()` chạy `SELECT * FROM documents` KHÔNG thấy row vừa INSERT (chưa commit) → 0 chunk. Plan này BẮT BUỘC `service.create` COMMIT NGAY sau INSERT documents, trước khi router add BackgroundTask. Comment trong code phải giải thích ordering này rõ ràng.

Purpose: INGEST-04/05 — upload + status tracking + A4 trigger.
Output: schemas documents, documents_service.create/get + trigger helper, documents router, integration test.
</objective>
