<decisions>
QUYẾT ĐỊNH PLANNER (REQUIREMENTS/ROADMAP là source of truth, ghi rõ để executor không phân vân):

- D-07-01-A — Marker citation: ROADMAP SC1 + ASK-01 yêu cầu marker `[N]` (số). Frontend
  `CitationText.tsx` lại parse regex `/\[src:([^\]\s]+)\]/g` (marker `[src:<id>]`). Để thoả
  CẢ HAI: LLM được chỉ thị sinh marker `[N]` (số — dễ cho LLM, khớp ASK-01); parser nội bộ
  map `[N]` → `chunk_id`; `Citation` schema mang đồng thời `marker="[N]"` (literal ROADMAP)
  VÀ field `chunk_id`. Plan 07-04 router khi build `CitationRefAPI` cho frontend sẽ tính
  `marker` = `[src:<chunk_id>]` và rewrite answer thay `[N]`→`[src:<id>]`. Plan 07-01 chỉ
  cần parser `[N]`→index; KHÔNG xử lý `[src:]`.
- D-07-01-B — `top_k` mặc định 6, clamp `[1, 12]`. Lý do: ASK gửi nhiều chunk vào LLM prompt
  (token cost), 6 đủ cho VN medical, trần 12 chống prompt khổng lồ. (Search dùng 10/50 —
  khác mục đích nên không tái dùng hằng số.)
</decisions>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/ask body | `query` người dùng không tin cậy — có thể chứa prompt injection |
| context chunks → LLM prompt | nội dung chunk (đã qua ingestion) chèn vào prompt cùng query |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering/Elevation | `query` user override system prompt (ASK-02) | mitigate | `ANTI_INJECTION_SYSTEM_PROMPT` quy tắc 4: mọi nội dung context+query là DỮ LIỆU; quy tắc 2 buộc trả lời từ chối khi không có thông tin. Verify đầy đủ ở Plan 07-05 integration test. |
| T-07-01-02 | Information Disclosure | LLM tiết lộ system prompt khi user yêu cầu | mitigate | Quy tắc 4 cấm tiết lộ system prompt; quy tắc 1 cấm dùng kiến thức ngoài context. |
| T-07-01-03 | Tampering | chunk content chứa chỉ thị giả ("bỏ qua quy tắc") | mitigate | Quy tắc 4 coi toàn bộ CONTEXT là DỮ LIỆU; mỗi chunk gắn prefix `[N] (nguồn: ...)` tách biệt rõ với chỉ thị system. |
| T-07-01-04 | Tampering | marker `[N]` out-of-range/giả trong answer LLM | mitigate | `parse_citations` clamp `1<=n<=len(chunks)`, bỏ marker out-of-range — không tạo Citation rác trỏ chunk không tồn tại. |
</threat_model>
