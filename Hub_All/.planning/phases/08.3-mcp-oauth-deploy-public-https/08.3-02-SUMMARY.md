## Implementation Notes

### Task 1 — ApiClient.login()
- `login(email, password)` gọi `POST /api/auth/login` body `{email, password}` — credential ủy thác hoàn toàn API Service (Argon2), MCP Service KHÔNG verify password (D-02 anti-pattern).
- Phân biệt 3 kết quả: login OK → dict `{access_token, refresh_token, expires_at, user}`; credential sai (401) → `None` (KHÔNG raise — kết quả hợp lệ của login form); lỗi hạ tầng (network/5xx/response không hợp lệ) → raise `ApiServerError`.
- KHÔNG log email/password — chỉ log status (T-08.3-08).

### Task 2 — MedinetOAuthProvider
- Lớp `MedinetOAuthProvider(OAuthAuthorizationServerProvider[AuthorizationCode, RefreshToken, AccessToken])` — implement đủ 9 method SDK theo chữ ký CHÍNH XÁC đọc từ `.venv/.../mcp/server/auth/provider.py` (SDK 1.27).
- Constructor inject tường minh `(store, api_client, issuer_url, access_token_ttl, refresh_token_ttl)` — KHÔNG tự khởi tạo singleton.
- `authorize()` sinh `txn = secrets.token_urlsafe(32)`, lưu pending qua `store.save_pending`, trả URL `f"{issuer_url}/login?txn={txn}"`. Lưu ý SDK: `AuthorizationParams` KHÔNG có field `code_challenge_method` (chỉ `code_challenge`) — provider hard-code `"S256"` khi lưu pending (MCP spec bắt buộc S256).
- `complete_authorization(txn, login_data)` — method bổ trợ public cho login callback: load pending → sinh code → `save_auth_code` bind downstream JWT access+refresh + user payload → `delete_pending` → trả `(code, redirect_uri, client_state)`.
- Token OAuth opaque (`secrets.token_urlsafe(32)` — RESEARCH.md A3). `exchange_authorization_code` xoá code sau exchange (single-use, T-08.3-05). `exchange_refresh_token` rotate cả access + refresh qua `store.update_oauth_token`.
- `load_access_token` reject token hết hạn (`expires_at < now()`) — verify mỗi tool call.
- `revoke_token` nhận `AccessToken` hoặc `RefreshToken` — xoá theo record (revoke cả cặp).
- KHÔNG log giá trị token/code/JWT/txn — chỉ log `client_id` + loại thao tác.

### Task 3 — Login form + wiring + lifespan
- `oauth/login.py`: `render_login_form(txn, error)` trả HTML tối giản (`<form method="post" action="/login/callback">`, input email/password, label tiếng Việt, escape XSS qua `html.escape`). `get_login_routes(provider, api_client)` dùng closure inject dependency vào 2 handler Starlette `Route`.
- Login callback: credential sai → render form lỗi "Sai tài khoản hoặc mật khẩu" (HTTP 200); `ApiClientError` (lỗi hạ tầng) → "Lỗi hệ thống, vui lòng thử lại sau"; `OAuthStoreError` (txn hết hạn) → "Phiên đăng nhập hết hạn, vui lòng kết nối lại". Login OK → `RedirectResponse` 302 kèm `?code=...&state=...` (bỏ `&state=` nếu `client_state` None).
- `server.py`: `FastMCP` wired `auth_server_provider` + `AuthSettings` (issuer + resource_server URL, `ClientRegistrationOptions(enabled=True, valid/default_scopes=["wiki"])`, `RevocationOptions(enabled=True)`, `required_scopes=["wiki"]`). SDK tự mount `/.well-known/oauth-authorization-server`, `/.well-known/oauth-protected-resource`, `/authorize`, `/token`, `/register`, `/revoke`. Route `/login` + `/login/callback` thêm sau.
- Lifespan compose: giữ `app.router.lifespan_context` gốc của SDK, bọc `_composed_lifespan` chạy `await store.init_schema()` lúc startup TRƯỚC khi vào lifespan SDK lồng bên trong → schema sẵn sàng trước request đầu, lifespan session-manager SDK được bảo toàn. KHÔNG dùng `asyncio.run` cho `init_schema` (2 `asyncio.run` còn lại trong `_close_client_atexit` chỉ là cleanup `aclose()` best-effort).
- `stateless_http=True` giữ nguyên (Pitfall 2).
