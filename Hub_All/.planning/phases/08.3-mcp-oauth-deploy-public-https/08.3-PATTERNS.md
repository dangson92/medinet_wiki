### `mcp_service/mcp_app/oauth/store.py` (store — SQLite persistence cho OAuth state)

**Analog:** `mcp_service/mcp_app/api_client.py` — mẫu class async độc lập + cây exception riêng + cleanup `aclose()`.

**Cây exception riêng cho tầng gọi** (`api_client.py` lines 24-41) — store nên có exception class gốc tương tự nếu cần phân biệt lỗi:
```python
class ApiClientError(Exception):
    """Lỗi gốc khi gọi API Service."""

class ApiUnauthorizedError(ApiClientError):
    """401 — API key thiếu, sai, hoặc đã thu hồi."""
# ... ApiForbiddenError / ApiBadRequestError / ApiServerError
```

**Lifecycle async + context manager** (`api_client.py` lines 51-64) — store dùng `aiosqlite` async, copy mẫu `aclose()` + `__aenter__`/`__aexit__`:
```python
async def aclose(self) -> None:
    """Đóng httpx client — giải phóng connection pool."""
    await self._client.aclose()

async def __aenter__(self) -> ApiClient:
    return self

async def __aexit__(self, exc_type, exc, tb) -> None:
    await self.aclose()
```

**Bảo mật khi log** (`api_client.py` lines 11-12, docstring) — store KHÔNG log giá trị token/JWT/code; chỉ log `client_id`/`status`/`error.code`. Đây là pattern T-08.2-01-I áp dụng toàn `mcp_app`.

**Đóng resource lúc thoát process** (`server.py` lines 258-278) — `atexit.register(...)` best-effort `aclose()`. Store cũng nên đăng ký cleanup tương tự khi `server.py` khởi tạo store singleton.

**Khởi tạo schema lúc startup** — RESEARCH.md §"Runtime State Inventory" yêu cầu 3 bảng (`oauth_clients`, `oauth_auth_codes` TTL ngắn, `oauth_tokens` chứa downstream JWT+refresh). Path file qua config `MCP_OAUTH_STATE_DB_PATH` (xem `config.py` mục dưới).

```python
@field_validator("api_base_url", mode="after")
@classmethod
def _validate_base_url(cls, value: str) -> str:
    stripped = value.rstrip("/")
    parsed = urlparse(stripped)
    if parsed.scheme not in ("http", "https"):
        raise ValueError("MCP_API_BASE_URL phải dùng scheme http hoặc https ...")
    if not parsed.netloc:
        raise ValueError("MCP_API_BASE_URL thiếu host")
    return stripped
```

## Shared Patterns

### Module docstring tiếng Việt + REQ-ID
**Source:** `mcp_app/server.py` lines 1-18, `mcp_app/auth.py` lines 1-12

### Bảo mật — KHÔNG log secret
**Source:** `mcp_app/api_client.py` lines 11-12 + 79-115, `mcp_app/server.py` lines 302-308
**Apply to:** `oauth/provider.py`, `oauth/store.py`, `oauth/login.py`, `api_client.py` (sửa)
KHÔNG log token / JWT / password / Authorization header / API key. Chỉ log `method` / `path` / `status_code` / `error.code` / `client_id`. Pattern T-08.2-01-I / T-08.2-03-I2 — anti-pattern bị RESEARCH.md liệt kê tường minh.

### Map lỗi tập trung, message sạch (không stack trace)

### Đọc header từ MCP Context

**Apply to:** `auth.py` (thêm `extract_oauth_token`)
`ctx.request_context.request.headers.get(...)` bọc trong `try/except (AttributeError, LookupError, TypeError)` → trả `None` thay vì raise.

### Validator config chống misconfiguration
**Source:** `mcp_app/config.py` lines 36-49 (`_validate_base_url`)

`@field_validator(..., mode="after")` ép scheme http/https, strip trailing slash, bắt buộc host. Issuer URL prod PHẢI `https` (Pitfall 6).
