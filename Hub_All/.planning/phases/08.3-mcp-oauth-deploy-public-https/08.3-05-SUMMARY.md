## Accomplishments

- **CR-01** — `claim_auth_code` đọc+xoá authorization code trong một bước nguyên tử (`DELETE ... RETURNING`); `exchange_authorization_code` claim TRƯỚC `save_token` nên dù `save_token` lỗi giữa chừng, code không còn trong store để replay (RFC 6749 §10.5).
- **CR-02** — `rotate_token` UPDATE định vị theo `refresh_token` caller cầm + trả `cursor.rowcount == 1`; `exchange_refresh_token` raise lỗi reuse khi `rowcount != 1` thay vì rotate nhầm/trả token mồ côi (RFC 6749 §10.4).
- **Issue 3** — guard `record["expires_at"] < self._now()` ngay sau claim trong `exchange_authorization_code` — code hết hạn bị từ chối kể cả khi provider được gọi trực tiếp, bỏ qua lớp kiểm hạn của route SDK.
- **WR-02** — `complete_authorization` từ chối pending transaction quá hạn (`created_at < now - _PENDING_TTL` 600s).
- **WR-03** — `revoke_token` tra cứu record nhất quán theo cả access lẫn refresh token rồi xoá theo `access_token` của record.
- **WR-04** — `complete_authorization` validate shape `login_data` sớm; `login_callback` bắt thêm `(KeyError, TypeError)` render lỗi hệ thống chung; `server._call_api` validate shape `new_pair` trước khi dùng — payload downstream thiếu field cho lỗi có kiểm soát thay vì HTTP 500 + stack trace.
- **WR-05** — `CREATE UNIQUE INDEX IF NOT EXISTS idx_oauth_tokens_refresh` áp được cho cả DB cũ lẫn mới — loại nhập nhằng `load_token_by_refresh`.

## Decisions Made

- **claim_auth_code dùng `DELETE ... RETURNING`** thay vì SELECT-rồi-DELETE — đọc+xoá nguyên tử trong một query, loại race window và bảo đảm code không thể replay nếu `save_token` lỗi giữa chừng.
- **Giữ `delete_auth_code` + `update_oauth_token` cũ** — provider chuyển sang dùng `claim_auth_code`/`rotate_token`, nhưng method cũ vẫn còn cho cleanup và để test cũ không vỡ. `delete_auth_code` vẫn dùng trong `cleanup_expired`-style path.
- **Guard `expires_at` TẠI `exchange_authorization_code`** — `claim_auth_code` cố ý KHÔNG kiểm hạn (chỉ claim); kiểm hạn đặt ở provider để bảo vệ cả đường gọi provider trực tiếp, không phụ thuộc lớp kiểm hạn của route SDK `/token`.
