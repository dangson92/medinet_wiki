# Phase 8.3 Plan 03: Đường danh tính OAuth token → downstream JWT cho tool call Summary

Hoàn tất MCP-02 (D-03/D-04) — nối OAuth token tới API Service: mỗi tool call resolve credential qua `_resolve_credential` (ưu tiên `Authorization: Bearer <oauth_token>` → verify `provider.load_access_token` → lấy downstream JWT của đúng user từ `OAuthStore`; fallback `X-API-Key` cho client local), rồi `_call_api` forward `Authorization: Bearer <JWT>` xuống API Service. Đóng Pitfall 4/5 — downstream JWT hết hạn giữa phiên OAuth dài: API trả 401 → `refresh_jwt()` đổi JWT mới qua `POST /api/auth/refresh` → lưu rotation CẢ access + refresh vào store → retry 1 lần. 3 tool body logic giữ nguyên; X-API-Key local hoạt động song song.

## Implementation Notes

### Task 1 — extract_oauth_token + Bearer JWT forwarding + refresh_jwt
- `auth.extract_oauth_token(ctx)` — đọc header `authorization`, parse `Bearer <token>` (case-insensitive scheme), trả `None` thay vì raise nếu thiếu/sai format (caller fallback X-API-Key được). Cùng cấu trúc try/except `(AttributeError, LookupError, TypeError)` như `extract_api_key`. `parts[1].strip()` chặn `"Bearer "` (token rỗng) → None.
- `ApiClient._request` — `api_key` thành optional + thêm `jwt: str | None`; có `jwt` → header `{"Authorization": f"Bearer {jwt}"}`, không jwt có api_key → `{"X-API-Key": api_key}`, thiếu cả hai → raise `ApiClientError`. `get`/`post` pass-through `jwt`. Logic unwrap envelope + map status giữ nguyên.
- `ApiClient.refresh_jwt(refresh_token)` — `POST /api/auth/refresh` body `{refresh_token}`; 200+success → dict `{access_token, refresh_token, expires_at, user}`; 401 → `None` (refresh token hết hạn/blacklist); lỗi hạ tầng → `ApiServerError`. KHÔNG log refresh token/JWT.

### Task 2 — Tool resolve OAuth token + retry-on-401
- `auth.Credential` — dataclass `frozen` giữ `kind` ("jwt"|"api_key"), `value` (JWT hoặc api_key), `oauth_token` (str|None — cần để cập nhật store khi rotation).
- `server._resolve_credential(ctx)` — có Bearer token → `provider.load_access_token` verify (T-08.3-11); None → `ToolError` MCP_UNAUTHORIZED; hợp lệ → `store.load_token` lấy `downstream_jwt` → `Credential(kind="jwt", ...)`. Không OAuth → `extract_api_key` → `Credential(kind="api_key", ...)`. Không có gì → `ToolError` MCP_UNAUTHORIZED.
- `server._call_api(cred, method, path, ...)` — nhánh `api_key` gọi thẳng (KHÔNG retry — X-API-Key không refresh được); nhánh `jwt` gọi với JWT, bắt `ApiUnauthorizedError` lần đầu → `store.load_token` lấy `downstream_refresh_token` → `client.refresh_jwt(...)`; None → re-raise (→ ToolError MCP_UNAUTHORIZED); có → `store.update_downstream_jwt` lưu đè CẢ access + refresh (Pitfall 5) → retry 1 lần với JWT mới. Helper `_jwt`/`_key` tách giá trị credential theo `kind`.
- 3 tool (`list_hubs`/`search_wiki`/`ask_wiki`) thay `require_api_key(ctx)` + `_get_client().get/post(..., api_key=)` bằng `cred = await _resolve_credential(ctx)` + `await _call_api(cred, ...)`. Logic dựng path/body + parse response GIỮ NGUYÊN.
- `conftest.py` — thêm fixture `mock_ctx_oauth` (header `Authorization: Bearer <token>`). Autouse `reset_api_client` đã reset `_oauth_store`/`_oauth_provider` từ Plan 02 — không cần sửa thêm.
- Test mới seed `OAuthStore` `:memory:` thật + `MedinetOAuthProvider` thật, `monkeypatch` `_get_oauth_store`/`_get_oauth_provider` singleton — verify đường resolve thật (không mock provider).
