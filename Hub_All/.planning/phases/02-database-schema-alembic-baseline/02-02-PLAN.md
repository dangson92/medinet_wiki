# Plan 02: SQLAlchemy models cho 9 bảng (`users`, `hubs`, `documents`, `chunks`, `audit_logs`, `usage_events`, `refresh_tokens`, `api_keys`, `settings`) + bảng join `user_hubs`

## Objective
Định nghĩa 9 SQLAlchemy ORM model + 1 join table cho toàn bộ entity M2 — gom theo domain (auth/hub/document/chunk/audit/usage/settings) trong `Hub_All/api/app/models/`. Mọi model inherit `Base` + 2 mixin từ Plan 01; mọi FK/CHECK/UNIQUE/index khai báo declarative để Plan 03 (Alembic) autogenerate đầy đủ ở Plan 04.

**Design choice (Claude's discretion):** Thêm bảng join `user_hubs` (many-to-many) thay vì `users.hub_assignments UUID[]` — lý do: (a) FK constraint enforce hub_id valid (UUID[] không có constraint), (b) query JOIN nhanh hơn `WHERE hub_assignments @> ARRAY[$1]`, (c) HUB-02 isolation Phase 5 dễ enforce qua JOIN. Tổng = **10 bảng** (9 chính + 1 join), khớp ROADMAP success criteria #1 ("8 table chính + settings = 9 table" + 1 join là design enhancement).
