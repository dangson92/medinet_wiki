## Quyết định kỹ thuật

### D-AUTH-01: Import path FastMCP trong mcp==1.9.4

**Tình huống:** RESEARCH.md ghi `from fastmcp.server.dependencies import get_http_headers` — nhưng `mcp==1.9.4` KHÔNG có module `fastmcp` riêng. FastMCP tích hợp trong `mcp.server.fastmcp`.

**Khám phá:**
- `pip show mcp` → không có standalone `fastmcp` package
- `get_http_headers` không tồn tại trong `mcp==1.9.4`
- FastMCP nằm tại: `from mcp.server.fastmcp import FastMCP`
- HTTP headers trong MCP tool đọc qua: `ctx.request_context.request.headers` (Starlette `Request` object được pass vào `ServerMessageMetadata.request_context` bởi `StreamableHTTPSessionManager`)

**Quyết định:** `authenticate_mcp_request(ctx: object, pool: asyncpg.Pool)` — `ctx` là MCP Context object; đọc header qua `ctx.request_context.request.headers.get("x-api-key")`. Pool là reserve param (giữ nhất quán với Pattern 4 RESEARCH.md).

**Impact:** Wave 2 (`server.py`) phải truyền `ctx` (MCP Context) vào `authenticate_mcp_request` — tool handler cần annotate `ctx: Context` trong signature để FastMCP inject tự động.

### D-AUTH-02: pool là reserve param trong authenticate_mcp_request

**Tình huống:** RESEARCH.md Pattern 4 thiết kế `_get_pool()` singleton. authenticate_mcp_request nhận `pool: asyncpg.Pool` để giữ nhất quán nhưng KHÔNG dùng pool trực tiếp (dùng `get_engine()` + `AsyncSession` cho ApiKeyService).

**Quyết định:** Giữ `pool` trong signature với `# noqa: ARG001` comment. Wave 2 có thể refactor nếu muốn dùng asyncpg raw thay SQLAlchemy cho auth.
