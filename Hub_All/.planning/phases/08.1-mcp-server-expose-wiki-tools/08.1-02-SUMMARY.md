## Quyết định kỹ thuật

### D-SERVER-01: mcp==1.27.1 thay vì 1.9.4

**Tình huống:** `pyproject.toml` khai báo `mcp>=1.9.4,<2`, nhưng `uv` resolve ra `mcp==1.27.1`. PLAN.md và RESEARCH.md viết theo API `mcp==1.9.4` — có nhiều điểm khác nhau quan trọng.

**Khác biệt API phát hiện khi thực thi:**

| API (1.9.4 theo plan) | Thực tế trong 1.27.1 |
|----------------------|---------------------|
| `mcp.http_app(path="/", stateless_http=True)` | `mcp.streamable_http_app()` (không có tham số) |
| `stateless_http=True` trong `http_app()` | `stateless_http=True` trong `FastMCP.__init__()` |
| `@mcp.tool` (không có `()`) | `@mcp.tool()` (bắt buộc có `()`) |
| `from fastmcp.utilities.lifespan import combine_lifespans` | Không tồn tại |
| `mcp.get_tools()` → dict | `mcp.list_tools()` → `list[MCPTool]` |

**Quyết định:** Adapt toàn bộ code theo API thực tế của `mcp==1.27.1`. Ghi rõ trong docstring và SUMMARY.

### D-SERVER-02: _composed_lifespan thay combine_lifespans

**Tình huống:** `combine_lifespans` không tồn tại trong package nào được install. RESEARCH.md Pattern 1 ghi `from fastmcp.utilities.lifespan import combine_lifespans` nhưng đây là API của `fastmcp` standalone (PyPI riêng), không có trong `mcp` tích hợp.

**Giải pháp:** Viết tay `_composed_lifespan` tương đương:
```python
@asynccontextmanager
async def _composed_lifespan(app: FastAPI) -> AsyncIterator[None]:
    async with lifespan(app):
        async with _mcp_lifespan(app):
            yield
```
`_mcp_lifespan = _mcp_app.router.lifespan_context` — là `lambda app: session_manager.run()` từ Starlette router. Thứ tự: lifespan cha startup trước → MCP session manager startup → yield → MCP shutdown → lifespan cha shutdown. Nhất quán với semantics của `combine_lifespans`.

**Tại sao không dùng `lifespan=lifespan`:** Không compose → `StreamableHTTPSessionManager` không có task group khởi động → lỗi `Task group is not initialized` khi client gọi (Cạm bẫy 1 RESEARCH.md).

### D-SERVER-03: @mcp.tool() bắt buộc có dấu ()

**Tình huống:** PLAN.md viết `@mcp.tool` (không có `()`). Trong `mcp==1.27.1`, `@mcp.tool` (không `()`) raise `TypeError: The @tool decorator was used incorrectly. Did you forget to call it? Use @tool() instead of @tool`.

**Fix:** Đổi tất cả decorator thành `@mcp.tool()`.

### D-SERVER-04: ctx: Context là tham số đầu tiên tool handler

**Tình huống:** `authenticate_mcp_request(ctx, pool)` yêu cầu `ctx` (Wave 1 auth.py). Trong `mcp==1.27.1`, `ctx: Context` annotation đặt ở bất kỳ vị trí nào trong tham số tool function → FastMCP inject tự động. Đặt đầu tiên cho rõ ràng.

**Kết quả:** Tool handler signature: `async def list_hubs(ctx: Context) -> HubList` — FastMCP nhận ra `Context` type annotation và inject vào trước khi gọi tool.
