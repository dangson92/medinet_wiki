### Gaps Summary

**1 gap partial** được xác định:

**CR-01: asyncio.create_task không track — usage log có thể mất khi exception**

`ask_wiki` trong `server.py` gọi `asyncio.create_task(log_usage_event(...))` với `# noqa: RUF006` nhưng không có error guard. Nếu `log_usage_event` raise exception (DB unavailable, pool timeout, etc.), exception bị silently dropped — không có log, không có retry, không có cảnh báo. Đây là deviation so với pattern HTTP router dùng FastAPI `BackgroundTasks` (đảm bảo task hoàn thành).

**Fix đề xuất (từ REVIEW.md CR-01 Option A — đơn giản nhất):**
```python
# Thay asyncio.create_task → await trực tiếp
# MCP tool không phải hot-path latency — vài ms thêm hoàn toàn chấp nhận
await log_usage_event(pool, user_id=str(user.user.id), ...)
```

Hoặc Option B (giữ non-blocking nhưng có error guard):
```python
async def _log_with_error_guard() -> None:
    try:
        await log_usage_event(pool, ...)
    except Exception as exc:
        logger.error("ask_wiki_usage_log_failed: %s", exc)

asyncio.create_task(_log_with_error_guard())
```

Gap này không block SC1/SC2/SC3/SC4 — chỉ ảnh hưởng SC5 (usage logging). Status phase là `human_needed` vì SC1/SC5 cần human verify, không phải `gaps_found`.
