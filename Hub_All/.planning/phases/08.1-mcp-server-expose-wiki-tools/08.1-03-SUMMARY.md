## Quyết định kỹ thuật

### D-TEST-01: Mock ctx thay vì patch get_http_headers

**Tình huống:** PLAN.md viết `monkeypatch "app.mcp.auth.get_http_headers"` — nhưng `get_http_headers` KHÔNG tồn tại trong `mcp==1.27.1` (xác nhận từ Wave 1/2 SUMMARY). `auth.py` đọc header qua `ctx.request_context.request.headers.get("x-api-key")`.

**Quyết định:**
- `test_missing_api_key_raises_tool_error` và `test_search_wiki_auth_gate`: Dựng `mock_ctx = MagicMock()` với `ctx.request_context.request.headers.get = MagicMock(return_value=None)` → truyền vào tool handler → `_get_api_key_from_ctx()` trả `None` → `ValueError` → `ToolError`.
- `test_invalid_api_key_raises_tool_error`: Monkeypatch `app.mcp.server.authenticate_mcp_request` tại module server để simulate key thu hồi — tránh cần mock cả `AsyncSession` + `ApiKeyService`.

### D-TEST-02: patch SearchService tại module gốc

**Tình huống:** PLAN ghi `patch("app.mcp.server.SearchService")` — nhưng `search_wiki` import `SearchService` LOCAL trong function body (`from app.services.search_service import SearchService  # noqa: PLC0415`). Mỗi lần hàm gọi, Python lookup lại từ `sys.modules["app.services.search_service"]` — không bind tại `app.mcp.server`.

**Quyết định:** Dùng `patch("app.services.search_service.SearchService")` — thay thế trong module gốc, mọi local import trong lần gọi tiếp theo đều thấy mock. `AskService` (import module-level) vẫn patch tại `app.mcp.server.AskService` đúng.

### D-TEST-03: Tool handler signature yêu cầu ctx

**Tình huống:** PLAN gọi `await mcp_server_module.list_hubs()` không có tham số. Thực tế signature: `list_hubs(ctx: Context)` — FastMCP inject `ctx` khi gọi qua MCP protocol nhưng KHÔNG inject khi gọi trực tiếp trong test.

**Quyết định:** Truyền `ctx = MagicMock()` làm tham số đầu tiên khi gọi trực tiếp. Khi `authenticate_mcp_request` bị mock hoàn toàn (các test tool behavior), ctx không cần thiết nhưng phải truyền để match signature. Khi test auth flow thật, mock ctx phải có đầy đủ `request_context.request.headers` structure.

### D-TEST-04: asyncio.create_task coroutine leak

**Tình huống:** Test `test_ask_wiki_returns_answer_with_citations` mock `asyncio` nhưng coroutine `log_usage_event(...)` được tạo và truyền vào `mock_asyncio.create_task` — nếu không đóng, Python GC phát `RuntimeWarning: coroutine was never awaited`.

**Quyết định:** Trong `_safe_create_task()` helper: gọi `coro.close()` ngay trước khi return `MagicMock()`. Áp dụng cho cả 2 test `ask_wiki`.

## Hướng dẫn kết nối Claude Code (.mcp.json)

Để kết nối Claude Code Desktop với MCP server Medinet Wiki, tạo `.mcp.json` tại thư mục project:

```json
{
  "mcpServers": {
    "medinet-wiki": {
      "type": "http",
      "url": "http://localhost:8080/mcp",
      "headers": {
        "x-api-key": "YOUR_API_KEY_HERE"
      }
    }
  }
}
```

Lưu ý:
- `YOUR_API_KEY_HERE`: thay bằng API key tạo qua endpoint `POST /api/api-keys` (Phase 5).
- API key phải thuộc user có quyền truy cập hub (viewer role trở lên).
- MCP server chạy trong process FastAPI — đảm bảo `api/` đang running trước khi kết nối.
- Tool hoạt động: `list_hubs`, `search_wiki`, `ask_wiki`.
