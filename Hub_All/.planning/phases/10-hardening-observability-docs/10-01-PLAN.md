Hub_All/api/app/middleware/request_id.py (hiện tại — sẽ mở rộng):
```python
REQUEST_ID_HEADER = "X-Request-Id"

class RequestIdMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next: RequestResponseEndpoint) -> Response:
        rid = request.headers.get(REQUEST_ID_HEADER) or str(uuid.uuid4())
        request.state.request_id = rid
        response = await call_next(request)
        response.headers[REQUEST_ID_HEADER] = rid
        return response
```

| Boundary | Description |

| Logger → stdout | Log JSON ghi ra stdout → Docker capture → ops dashboard. PII leak risk nếu log raw email/password/token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Information Disclosure | logging_config.py + middleware log "request_completed" | mitigate | Log `request_id` (UUID) + `user_id` (UUID) + `hub_id` (UUID) — KHÔNG bao giờ log raw `Authorization` header, raw email, password, X-API-Key value. Verify qua code review tasks (grep `logger.info` trong app/ không khớp pattern `email=|password=|X-API-Key|Authorization`) |
| T-10-01-02 | Spoofing | Client gửi `X-Request-Id` malicious | accept | Echo nguyên trị (giữ semantic Go behavior). Risk thấp — request_id chỉ dùng correlation log, không có quyết định auth/business |
