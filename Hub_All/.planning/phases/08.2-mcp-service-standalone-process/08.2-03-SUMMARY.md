# Phase 8.2 Plan 03: Lõi MCP Service — FastMCP + 3 tool gọi API qua HTTP Summary

Hiện thực lõi MCP Service như process độc lập: `auth.py` trích `X-API-Key` từ HTTP header MCP request (bỏ verify DB — verify chuyển sang API Service), `server.py` dựng FastMCP server với 3 tool `list_hubs` / `search_wiki` / `ask_wiki` gọi API Service qua HTTP bằng `ApiClient` (Plan 01), map lỗi 401/403/400/5xx sang `ToolError`, và thêm entrypoint `build_asgi_app()` + `main()` để MCP Service chạy standalone qua `python -m mcp_app.server` trên port riêng (default 8190). Hoàn tất đảo decision D-04 ở tầng tool — MCP tool gọi API qua HTTP thay vì import service layer in-process.

## Điểm chính

- **auth.py:** `extract_api_key(ctx)` đọc `ctx.request_context.request.headers.get("x-api-key")`, bọc `try/except (AttributeError, LookupError, TypeError)` → trả `None` nếu thiếu. `require_api_key(ctx)` raise `ToolError("MCP_UNAUTHORIZED: ...")` nếu thiếu key. KHÁC Phase 8.1: bỏ hoàn toàn verify DB (`asyncpg`/`ApiKeyService`/`sqlalchemy`) — verify do API Service làm khi nhận request. KHÔNG bao giờ log giá trị key.
- **server.py — 3 tool:** `mcp = FastMCP("Medinet Wiki MCP Service", stateless_http=True)`. `_get_client()` lazy-init `ApiClient` singleton từ config (tái dùng `httpx.AsyncClient` cho connection pooling).
  - `list_hubs` → `GET /api/hubs` (params `per_page=100`), map `data["items"]` → `HubList`.
  - `search_wiki` → single-hub `POST /api/search` (`hub_ids=[hub_id]`), cross-hub `POST /api/search/cross-hub` (`hub_ids=None`); clamp `top_k` [1,20].
  - `ask_wiki` → single-hub `POST /api/ask` (`hub_id`), cross-hub `POST /api/search/answer` (`hub_ids=None` — KHÔNG dùng `/api/ask/cross-hub` vì endpoint đó yêu cầu `hub_ids` non-rỗng). Helper `_to_citations` xử lý 2 shape field citation khác nhau (`chunk_id`/`content_snippet` vs `id`/`snippet`).
- **Map lỗi an toàn:** `_map_api_error` map `ApiUnauthorizedError`→`MCP_UNAUTHORIZED`, `ApiForbiddenError`→`HUB_ACCESS_DENIED`, `ApiBadRequestError`→`INVALID_QUERY`, `ApiServerError`→`API_ERROR`. Message ngắn gọn từ envelope error — KHÔNG đính kèm stack trace (T-08.2-03-I, Test 9 verify).
- **Entrypoint standalone:** `build_asgi_app()` trả `mcp.streamable_http_app()` (Starlette app, tự gắn lifespan session manager — không cần compose thủ công như `api/app/main.py`). `main()` chạy `uvicorn.run` với `service_host`/`service_port` từ config (KHÔNG hardcode). `atexit` hook best-effort đóng `ApiClient` lúc thoát. Log khởi động KHÔNG chứa `api_key`.
- **Không import service layer:** `server.py` chỉ import `mcp`, `mcp_app.*` — `grep` xác nhận KHÔNG có `import asyncpg`, `import app.`, `log_usage_event`, `AskService`, `SearchService` (chuỗi xuất hiện chỉ trong docstring giải thích, không phải import statement).

## Threat mitigations áp dụng

- **T-08.2-03-S (Spoofing):** `require_api_key` raise `ToolError(MCP_UNAUTHORIZED)` khi thiếu header; API 401 → `ApiUnauthorizedError` → `ToolError(MCP_UNAUTHORIZED)`. Tool KHÔNG chạy logic khi auth fail (Test 6/7).
- **T-08.2-03-E (Elevation of Privilege):** Hub isolation enforce API-side; `_map_api_error` map `ApiForbiddenError` → `ToolError(HUB_ACCESS_DENIED)`. MCP chỉ truyền `hub_id`/`hub_ids` xuống API (Test 8).
- **T-08.2-03-I (Information Disclosure — ToolError):** `_map_api_error` tạo message ngắn gọn từ envelope error, KHÔNG leak stack trace (Test 9 verify `Traceback`/`File "` không có trong message).
- **T-08.2-03-I2 (Information Disclosure — log):** Log khởi động chỉ `host`/`port`/`api_base_url`; `extract_api_key`/`require_api_key` KHÔNG log giá trị key.
- **T-08.2-03-D (DoS):** `ApiClient` (Plan 01) đặt `httpx` timeout từ config — tool không treo vô hạn.
