<objective>
Cho phép MCP Service (tool `list_hubs`) lấy danh sách hub qua HTTP. Hiện `GET /api/hubs` là admin-only (`require_role("admin")`) nên API key của user non-admin không gọi được. Plan này sửa `GET /api/hubs` để: (1) chấp nhận xác thực qua `X-API-Key` HOẶC JWT (`get_api_key_or_jwt`), (2) scope kết quả theo quyền của user — non-admin chỉ thấy hub được assign, admin thấy mọi hub active. Các endpoint hubs còn lại (POST/PUT/PATCH/stats) GIỮ NGUYÊN admin-only.

Mục đích: MCP Service gọi `GET /api/hubs` qua HTTP để cung cấp `list_hubs` — hub isolation enforce API-side (đáp ứng SC2 + SC3 của phase: "hub isolation vẫn enforce API-side"). Đây là điều kiện tiên quyết cho Plan 03 (MCP server tool `list_hubs`).
Output: `GET /api/hubs` hoạt động với X-API-Key, kết quả scope đúng theo role; test unit verify cả 2 nhánh (admin / non-admin).
</objective>
