## Điểm chính

- **conftest.py:** 3 fixture chung. `mock_ctx` là factory `_make(api_key=...)` — dựng `ctx` MagicMock với `request_context.request.headers` là **dict thật** (`{"x-api-key": key}` hoặc `{}` khi `api_key=None`) để `extract_api_key` đọc đúng giá trị/None (MagicMock sẽ trả Mock cho mọi `.get(...)`). `reset_api_client` (autouse) đặt `server._api_client = None` trước + sau mỗi test — tránh singleton leak base_url giữa test. `api_base_url` set env `MCP_API_BASE_URL` qua `monkeypatch` + `get_settings.cache_clear()` để `_get_client()` khởi tạo `ApiClient` với base URL khớp respx mock.
- **test_integration_tools.py — 9 test, mock tầng HTTP transport:** Khác `test_server.py` (Plan 03) mock ở tầng `ApiClient` (`_get_client`), plan này mock ở tầng HTTP (`respx.get/post`) — verify cả `ApiClient` (unwrap envelope, map status→exception) + tool (gọi đúng path, build body, map exception→`ToolError`) cùng nhau qua đúng boundary HTTP. Mỗi test dùng `@respx.mock`; helper `_envelope_ok` / `_envelope_err` dựng envelope `{success,data,error,meta}` khớp shape thật API Service.
  - Test 1 (`list_hubs` happy): `GET /api/hubs` → 2 hub; verify header `X-API-Key` đúng giá trị.
  - Test 2 (`search_wiki` single-hub): `POST /api/search`; verify body `hub_ids=["h1"]`, `query`, `top_k`.
  - Test 3 (`search_wiki` cross-hub): `POST /api/search/cross-hub`; verify body `hub_ids=None`.
  - Test 4 (`ask_wiki` single-hub): `POST /api/ask`; answer giữ marker `[1]`, citations field `chunk_id`/`content_snippet` map đúng.
  - Test 5 (`ask_wiki` cross-hub): `POST /api/search/answer` (verify route `/api/ask/cross-hub` KHÔNG được gọi); citations field `id`/`snippet`.
  - Test 6 (auth fail thiếu key): `ctx` không header → `ToolError MCP_UNAUTHORIZED`; verify route HTTP **KHÔNG** được gọi (tool fail trước khi gọi HTTP — `assert_all_called=False`).
  - Test 7 (API 401): envelope `INVALID_API_KEY` → `ToolError MCP_UNAUTHORIZED`.
  - Test 8 (hub isolation 403): envelope `FORBIDDEN` → `ToolError HUB_ACCESS_DENIED`.
  - Test 9 (API 5xx): envelope `LLM_FAILED` → `ToolError API_ERROR`; message KHÔNG chứa `Traceback` / `File "`.
  - Test 1/6/7/8 đánh marker `@pytest.mark.critical` (auth/hub isolation/happy path lõi).
- **pyproject.toml:** Khai báo marker `critical` trong `[tool.pytest.ini_options].markers` — bắt buộc vì `addopts` đã bật `--strict-markers`.
- **README:** Thêm 3 mục — "Chạy test" (bộ test respx, không cần API Service/DB thật, liệt kê 5 file test), "Kiến trúc sau Phase 8.2" (sơ đồ `Frontend ─► API Service` + `LLM Agent ─► MCP Service ─► API Service (HTTP)`, ghi đảo D-04), "UAT thủ công — SC4 usage_events" (4 bước SELECT count trước/sau). - **T-08.2-05-S (Spoofing):** Test 6 verify thiếu key → `ToolError MCP_UNAUTHORIZED` và tool fail TRƯỚC khi gọi HTTP (`assert not route.called`); Test 7 verify API 401 → `ToolError MCP_UNAUTHORIZED`.

- **T-08.2-05-I (Information Disclosure):** Test 9 verify `ToolError` từ API 5xx có message sạch — không `Traceback`, không `File "`.
