# Phase 8.2 Plan 04: Gỡ MCP server in-process khỏi API Service Summary

Gỡ sạch MCP server in-process khỏi API Service — hoàn tất đảo decision D-04. Sau khi Plan 03 chứng minh MCP Service chạy standalone được như process độc lập (`mcp_service/`), plan này gỡ toàn bộ wiring MCP khỏi `api/app/main.py` (block inject `mcp_set_pool` trong `lifespan()`, `_mcp_server`/`_mcp_app`/`_mcp_lifespan`/`_composed_lifespan` trong `create_app()`, `app.mount("/mcp", ...)`), xoá hẳn package `api/app/mcp/` (4 file) và thư mục test `api/tests/unit/mcp/` (4 file), gỡ dependency `mcp>=1.27.0,<1.28` khỏi `pyproject.toml`. API Service nay không còn biết gì về MCP — MCP là process riêng gọi API qua HTTP. SC1 của phase đạt: `main.py` không còn `/mcp` + `_composed_lifespan`.

### Ghi chú thực thi (không phải deviation)

- **Cài dependency package (Rule 3 — blocking issue):** Môi trường `api/` chưa cài dependency nào (`fastapi`/`asyncpg`/`alembic`... đều thiếu) — `pytest` không chạy được. Đã `pip install -e ".[dev]"` để chạy regression test (thiết lập môi trường thường lệ, không sửa code). Xác nhận `mcp` KHÔNG được cài lại — đúng như mong đợi sau khi gỡ khỏi `pyproject.toml`.
- **UnicodeDecodeError khi `ast.parse`:** `python -c "open('app/main.py')"` mặc định dùng codec `cp1252` trên Windows → lỗi đọc ký tự tiếng Việt UTF-8. Đổi sang `open(..., encoding='utf-8')` — vấn đề công cụ verify, không phải lỗi code.
- **Cảnh báo LiteLLM botocore:** `create_app()` log warning `could not pre-load bedrock-runtime ... No module named 'botocore'` — dependency optional của LiteLLM, không liên quan plan này, pre-existing. Không xử lý (ngoài phạm vi — SCOPE BOUNDARY).
- **Cảnh báo CRLF của Git:** Windows `core.autocrlf` cảnh báo `LF will be replaced by CRLF` khi commit — vô hại.
